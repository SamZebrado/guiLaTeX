# guiLaTeX 总装版公开总览

**编制**：外务大臣
**日期**：2026-04-19（最新裁决更新）
**阶段**：v1-candidate · 第一阶段收官区

---

## 1. 项目一句话简介

guiLaTeX 是一个可视化 LaTeX 编辑器，支持 PDF 元素级编辑，通过 Export IR 中间层实现 LaTeX 代码生成。Web 端为当前主展示线，Qt 端为原型阶段，Core 为共享导出支撑层。

## 2. 当前阶段

**v1-candidate — 第一阶段收官区**

Web 端已完成核心功能验证，是当前对外主展示线。Qt 端核心闭环已跑通，但多项功能尚未收口，不作为主完成判据。Core 导出内核稳定，作为 Web 和 Qt 的共享支撑层。

**这不是最终完整版。**

## 3. 三端定位

| | Web | Qt | Core |
|---|---|---|---|
| **定位** | **当前主展示线** | 原型阶段 | 支撑层 |
| **状态** | 独立应用候选版 | Roundtrip 已跑通，多项未收口 | 已封板 |
| **验证层级** | 浏览器级 | 脚本级（部分桌面级） | 脚本级 |
| **是否独立产品线** | 是 | 否（原型） | 否（共享内核） |

## 4. Web 当前状态（主展示线）

Web 已经是**独立应用候选版**，是当前对外主展示线。

**已浏览器级验证**：UI 布局、文本元素 CRUD、多选与旋转、图层管理、IR/JSON 导出、PDF 导出（浏览器打印）、项目保存/打开（JSON）、复制粘贴、点击瞬移修复

**仅 Bridge/脚本层验证**：LaTeX 导出、LaTeX 导入、跨端导回（Qt tex → Web）

**Blocked**：浏览器内原生 LaTeX 处理、部分高级字段（颜色、对齐等）、直接打开 .tex 文件

**演示说明**：演示录屏由用户人工录制完成，非自动生成。

## 5. Qt 当前状态（原型阶段）

Qt 核心闭环已跑通，但**不作为当前主完成判据**。

**已验证**：元素选择与属性编辑、旋转（仅通过配置面板）、复制/粘贴、导出/导入 conforming LaTeX、roundtrip 主闭环、Core 集成

**未收口问题**：
- ⚠️ 旋转仅能通过配置面板调节，无直接旋转控件
- ⚠️ 图层功能在编辑界面中未有效体现
- ⚠️ 导出 PDF 会造成闪退
- ⚠️ font_family Core gap（中文字体硬编码为 SimSun、英文字体硬编码为 Times New Roman）
- ❌ 保存项目、打开项目、变换菜单功能未实现

## 6. Core 当前状态（支撑层）

Core 是 Web 和 Qt 的**共享导出内核**，不是独立产品线。

- IR Schema 已定义
- LaTeX 导出器已实现
- Golden sample + 回归样本齐备
- 已知限制：font_family 映射硬编码

## 7. 当前最适合公开展示的能力

1. **Web 端独立编辑体验**：纯浏览器打开，零依赖，文本 CRUD、多选旋转、图层管理
2. **IR/JSON 导出**：Web 端支持，可查看结构化文档模型
3. **Playwright 自动化回归**：Web 端有稳定的自动化测试证据链
4. **LaTeX roundtrip**：Qt 端 roundtrip（导出 → 重新导入 → 差异对比），脚本级验证
5. **跨端导回**：Qt 导出的 conforming tex 可通过 bridge 导入 Web

## 8. 当前最不适合说满的能力

1. ❌ "Qt 与 Web 同等级完成" — Qt 仍有多项功能未收口
2. ❌ "浏览器内原生支持 LaTeX 导入/导出" — 实际依赖 Python bridge
3. ❌ "PDF 导出走 Core→tex→编译路径" — Qt 端 PDF 导出会闪退
4. ❌ "字体选择完全保真" — Core gap，中文字体硬编码
5. ❌ "自动生成演示素材" — 演示录屏由用户人工完成
6. ❌ "Core 是独立产品线" — Core 是 Web/Qt 的共享支撑层
7. ❌ "所有功能都已实现" — 两端均有 blocked 项
