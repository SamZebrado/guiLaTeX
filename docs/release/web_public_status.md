# Web 端公开状态

**更新日期**：2026-04-19（最新裁决更新）
**定位**：**当前主展示线**

---

## 总体评价

Web 端是 guiLaTeX 当前**最成熟、最适合对外展示**的端。纯浏览器应用，零依赖，直接打开 HTML 文件即可使用。已通过浏览器级验证的核心功能覆盖了完整的编辑-导出链路。

**这不是最终完整版。**

## 已浏览器级验证的功能

| 功能 | 状态 | 验证方式 |
|------|------|---------|
| UI 布局渲染 | ✅ | 手动 + Playwright 回归 |
| 文本元素 CRUD（创建/编辑/删除） | ✅ | 手动 + Playwright 回归 |
| 多选（框选） | ✅ | 手动 + Playwright 回归 |
| 旋转（多选旋转 45°） | ✅ | 手动 + Playwright 回归 |
| 图层管理 | ✅ | 手动验证 |
| IR/JSON 导出 | ✅ | 手动 + JSON 结构验证 |
| PDF 导出（浏览器打印） | ✅ | 手动验证 |
| 项目保存/打开（JSON） | ✅ | 手动验证 |
| 复制/粘贴 | ✅ | 手动验证 |
| 点击瞬移修复 | ✅ | Playwright 自动化锁定 |

## 仅 Bridge/脚本层验证的功能

| 功能 | 状态 | 说明 |
|------|------|------|
| LaTeX 导出 | ⚠️ Bridge | 通过 Python bridge 脚本调用 Core 导出器 |
| LaTeX 导入 | ⚠️ Bridge | 通过 Python bridge 脚本调用 Core 解析器 |
| 跨端导回（Qt tex → Web） | ⚠️ Bridge | Qt 导出 conforming tex → bridge 转换 → Web 导入 |

## Blocked / 未实现

| 功能 | 状态 | 说明 |
|------|------|------|
| 浏览器内原生 LaTeX 处理 | ❌ | 需要 Python 环境，非浏览器原生能力 |
| 部分高级字段 | ❌ | 颜色、对齐方式等字段尚未完整支持 |
| 直接打开 .tex 文件 | ❌ | 需要先通过 bridge 转换为 IR |

## 演示说明

- Web 端演示录屏由用户**人工录制**完成
- 自动录屏任务未真正成功，不声称"自动生成演示素材"为已成熟能力
- Playwright 自动化测试用于回归验证，不用于生成演示素材

## 对外一句话

> Web 端是 guiLaTeX 的当前主展示线：纯浏览器应用，零依赖，文本编辑、多选旋转、图层管理、IR/PDF 导出均已浏览器级验证。LaTeX 功能依赖 Python bridge。
