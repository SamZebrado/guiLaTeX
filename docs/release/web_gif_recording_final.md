# Web GIF 录制最终脚本

## 1. 录制准备

### 1.1 环境准备
- 打开浏览器（推荐 Chrome 或 Firefox）
- 调整浏览器窗口宽度为 1000px 左右
- 确保桌面整洁，无隐私内容
- 准备屏幕录制工具（如 QuickTime Player、OBS Studio 或 GIPHY Capture）

### 1.2 文件准备
- 确保 `release_public/web_prototype/index.html` 文件存在
- 测试页面加载是否正常

## 2. 主 GIF 录制脚本：Web 核心编辑链

### 2.1 录制目标
展示 guiLaTeX Web 应用的核心编辑功能，包括添加文本、编辑内容、拖动元素、多选和旋转。

### 2.2 操作步骤

| 步骤 | 操作 | 时长 | 说明 |
|------|------|------|------|
| 1 | 打开 `release_public/web_prototype/index.html` | 2秒 | 展示页面加载过程 |
| 2 | 展示界面 | 1秒 | 停顿展示整体布局 |
| 3 | 添加文本元素 | 1秒 | 点击添加文本按钮（如果有）或使用现有文本元素 |
| 4 | 编辑文本内容 | 2秒 | 在右侧属性面板中编辑文本为 "Hello guiLaTeX" |
| 5 | 拖动文本元素 | 1秒 | 拖动文本到新位置 |
| 6 | 添加第二个文本元素 | 1秒 | 添加第二个文本元素 |
| 7 | 多选元素 | 1秒 | 启用多选模式并选择两个文本元素 |
| 8 | 旋转元素 | 2秒 | 使用旋转滑块将元素旋转 45° |
| 9 | 展示结果 | 1秒 | 停顿展示旋转后的结果 |
| 10 | 导出 IR | 2秒 | 点击 "导出 IR" 按钮，展示导出功能 |

### 2.3 录制要求
- 总时长：10–15 秒
- 录制区域：仅浏览器窗口
- 分辨率：宽度 900–1100 px
- 鼠标动作：缓慢、流畅
- 每步停顿：0.5–1 秒
- 文件大小：控制在 5–8 MB 内

### 2.4 输出文件
- `release_public/assets/web_demo.gif`
- 备选：`release_public/assets/web_demo.mp4`

## 3. 补充 GIF 录制脚本：Web 导出流程

### 3.1 录制目标
展示 guiLaTeX Web 应用的项目保存、打开和 IR 导出功能。

### 3.2 操作步骤

| 步骤 | 操作 | 时长 | 说明 |
|------|------|------|------|
| 1 | 打开 `release_public/web_prototype/index.html` | 2秒 | 展示页面加载过程 |
| 2 | 编辑元素 | 2秒 | 选择一个文本元素并编辑其内容 |
| 3 | 保存项目 | 2秒 | 点击 "保存项目" 按钮，展示下载的 JSON 文件 |
| 4 | 重新打开项目 | 2秒 | 点击 "打开项目" 按钮，选择之前保存的 JSON 文件 |
| 5 | 导出 IR | 2秒 | 点击 "导出 IR" 按钮，展示导出的 JSON 文件 |
| 6 | 展示结果 | 1秒 | 停顿展示导出结果 |

### 3.3 录制要求
- 总时长：10–12 秒
- 录制区域：仅浏览器窗口
- 分辨率：宽度 900–1100 px
- 鼠标动作：缓慢、流畅
- 每步停顿：0.5–1 秒
- 文件大小：控制在 5–8 MB 内

### 3.4 输出文件
- `release_public/assets/web_export_flow.gif`

## 4. 录制技巧

### 4.1 画面优化
- 确保浏览器窗口干净，无多余标签页
- 关闭浏览器地址栏和书签栏，只显示内容区域
- 选择中性的背景色，确保文本清晰可见

### 4.2 操作优化
- 提前练习操作步骤，确保流畅
- 鼠标移动缓慢、平稳
- 点击位置准确，避免误操作
- 每步操作后停顿一下，让观众看清楚

### 4.3 后期处理
- 如果使用视频录制工具，可后期转换为 GIF
- 调整 GIF 帧率为 15–20 fps，平衡文件大小和流畅度
- 裁剪多余的边框，只保留浏览器内容区域
- 压缩 GIF 文件，确保大小在 5–8 MB 内

## 5. 备选方案

如果无法生成 GIF，可以选择以下备选方案：

### 5.1 方案 A：MP4 视频
- 录制为 MP4 格式
- 输出文件：`release_public/assets/web_demo.mp4`
- 分辨率：1000px 宽度
- 时长：10–15 秒

### 5.2 方案 B：Storyboard PNG
- 逐帧截图
- 输出目录：`release_public/assets/web_demo_storyboard/`
- 命名格式：`01.png`, `02.png`, ..., `10.png`
- 附带 README.md 说明

### 5.3 方案 C：静态截图
- 关键步骤截图
- 输出目录：`release_public/assets/web_screenshots/`
- 包含：界面展示、编辑操作、多选旋转、导出功能
