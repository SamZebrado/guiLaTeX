# Web Agent Guard - Web 线开发规范

## 版本信息
- **版本**: 1.2
- **最后更新**: 2026-04-12
- **更新者**: Web Agent

## 核心规则

### 1. 最新用户反馈优先原则
**规则ID**: `latest-user-feedback-overrides`
- 最新用户手测结果高于上一轮的判断
- 用户说“还是会瞬移/还是不对”，就不能继续沿用“已修复”
- 必须以用户实际测试结果为准，而非代码层面的修改

### 2. 证据优先修复原则
**规则ID**: `evidence-first-fix`
- 代码修改不等于问题修复
- 结论必须明确区分：
  - `code-level changed`: 代码已修改，但未验证
  - `executed and verified`: 已执行并验证通过
  - `still needs user verification`: 仍需用户手动验证

### 3. GUI 端到端验证保障
**规则ID**: `gui-e2e-guard`
- 浏览器自动化或真实交互模拟，才接近用户级自动测试
- 页面能打开、脚本能跑、console 无报错，不等于用户问题已解决
- 必须进行真实用户操作路径的测试

### 4. 真实路径测试原则
**规则ID**: `real-path-test-only`
- 测试必须优先覆盖用户真实路径
- 尤其要覆盖：
  - 点击对象不拖动
  - 点击对象后立即拖动
  - 点击对象不同位置（中心、右下、边缘）
  - 未旋转对象
  - 已旋转对象
- 不要用简化测试替代真实交互链路

### 5. 手动操作需求声明
**规则ID**: `manual-action-needed`
- 如果需要用户手动验证、截图、切换浏览器、确认字体，最后必须单列 `MANUAL ACTION NEEDED`
- 明确列出需要用户操作的步骤和预期结果

### 6. UI 语义契约
**规则ID**: `ui-semantic-contract`
- 打开 / 保存 / 导出 / 图层 / 多选 / 复制粘贴 / 旋转 等文案必须和真实行为一致
- 不允许文案超前于功能

### 7. 字体安全默认原则
**规则ID**: `font-safety-default`
- 默认字体列表只保留开源 / 免费可商用字体
- 不要把 Microsoft YaHei / PingFang SC / Arial 这类放进默认安全列表
- 如果未来允许系统字体，必须单独标注“需用户自行确认授权”
- 推荐默认字体：
  - Noto Sans SC
  - Source Han Sans SC
  - Inter
  - Noto Sans
  - Sans Serif

### 8. 提示路由门控
**规则ID**: `prompt-route-gate`
- 先判断当前 prompt 是否属于 Web 线
- 如果 prompt 明显属于 Qt 线，必须返回 ROUTE_MISMATCH，不要继续执行
- 如果是 mixed，只执行 Web 相关部分，并明确忽略了哪些非 Web 部分

### 9. 阻塞后打包原则
**规则ID**: `blocked-then-pack`
- 同一个核心交互问题连续多轮没压实，就明确 blocked，并整理交接材料，而不是继续口头宣布成功
- 遇到阻塞时，应提供清晰的问题描述和可能的解决方案

## 核心功能验证标准

### 8.1 点击对象
- ⚠️ 点击后对象不瞬移（当前重点修复）
- ⚠️ 点击后立即拖动时，对象与光标关系正常
- ⚠️ 旋转后点击命中测试正确
- ⚠️ transform-origin 计算正确

### 8.2 多选功能
- ⚠️ 多选复选框真正生效（当前重点修复）
- ⚠️ 勾选后能保留多个 selected 对象
- ⚠️ 能框体/轮廓体现多选
- ⚠️ 整体移动作用于多个对象

### 8.3 旋转功能
- ⚠️ 旋转入口明确可见（当前重点修复）
- ⚠️ 旋转手柄或按钮可交互
- ⚠️ 旋转后对象位置和大小正确

### 8.4 字体设置
- ⚠️ 默认字体列表只包含开源/免费可商用字体（当前重点修复）
- ⚠️ requested_font_stack 明确
- ⚠️ computed_font_family 正确

## 验证流程

### 9.1 自动测试（如果可用）
1. 页面加载
2. 点击一个对象
3. 确认点击后不瞬移
4. 勾选多选模式
5. 选中至少两个对象
6. 执行一次整体移动
7. 找到并使用一次旋转入口（如果本轮实现）
8. 保存一张截图

### 9.2 手动测试路径
1. 打开页面
2. 点击任意对象，确认不瞬移
3. 勾选多选模式
4. 点击多个对象，确认都被选中
5. 拖动选中对象，确认所有选中对象都移动
6. 找到旋转入口并使用（如果本轮实现）
7. 检查字体下拉列表，确认只包含开源字体

## 证据收集
- 保存页面截图到 `docs/contest_evidence/screenshots/`
- 记录控制台日志
- 保存测试过程录屏（如果可能）
- 记录操作步骤和结果

## 结论输出格式
- 明确区分已修复、部分缓解、未修复
- 提供具体证据支持
- 列出需要用户手动验证的项目
- 提供 MANUAL ACTION NEEDED 部分

## 版本控制
- 每次更新技能文档时，记录版本号和修改内容
- 保持与项目实际状态同步

## 适用范围
- 本规范适用于 Web 线所有开发和测试活动
- 所有 Web 相关的修复和功能实现必须遵循本规范
- 验证结果必须符合本规范的要求
